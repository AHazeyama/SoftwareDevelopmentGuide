## Sofutware Development Guide
ソフトウェア開発を効率的かつ一定品質で進めるための開発ガイドです。  
要件定義から設計、実装、テスト、リリースまでの標準手順をまとめています。  
個人開発だけでなく、小規模〜中規模の受託開発や業務システム開発で再利用できることを目的としています。
## 要件定義
業務内容やシステム化の目的を整理し、必要な機能・制約・対応範囲を明確化するためのガイドです。  
ヒアリングシートや業務フロー、機能一覧など、開発の土台となる成果物を作成します。
## 基本設計
システム全体の構成や画面、データ構造、機能の関連性を整理する工程です。  
実装前にシステム全体の設計方針を明確にし、開発・保守しやすい構成を目指します。
## 詳細設計 (コーディング規約)
クラス設計、関数設計、命名規則、ディレクトリ構成など、実装時のルールを定義します。  
可読性・保守性・再利用性を重視したコーディング標準をまとめています。
## UI設計
操作性・視認性・一貫性を重視したユーザーインターフェース設計のガイドです。  
画面レイアウト、配色、ボタン配置、メッセージ表示など、ユーザーが迷わず操作できる設計を目指します。
## ドキュメント作成
ソースコードや設計情報を保守しやすい形で文書化するためのガイドです。  
Doxygenを用いたAPIリファレンスの生成や、MarkdownによるREADME・設計書の作成方法など、継続的なドキュメント管理についてまとめています。  
## Git運用
GitおよびGitHubを利用したソースコード管理方法をまとめています。  
ブランチ運用、コミットルール、リリース管理など、継続的な開発を支える運用手順を定義します。
## 開発環境毎のテンプレート
言語・開発環境ごとの標準構成や初期設定をまとめています。
新規プロジェクトを短時間で立ち上げ、同じ品質で開発を開始できることを目的としています。
### C#（Visual Studio）
&emsp; Visual StudioによるWPFアプリケーション開発を中心に、プロジェクト作成、標準構成、  
&emsp; 単体EXE作成、Git運用までをまとめています。
### Python（Vim / Visual Studio）
&emsp; Python開発に必要な環境構築、ディレクトリ構成、テンプレート、ビルド、Git管理までをまとめています。
## Docker構築
Dockerを利用した開発環境の構築手順をまとめています。  
環境差異を最小限に抑え、再現性の高い開発環境を構築することを目的としています。
## リリース手順
アプリケーションのビルドから配布までの手順をまとめています。
動作確認、成果物作成、GitHub Releasesへの公開など、安定したリリースを行うための標準手順を定義します。
