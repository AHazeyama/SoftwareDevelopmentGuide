# Overview
基本設計の内容に基づいて、プログラムの実装および単体テストを円滑かつ正確に行うための詳細な仕様を定義します。
開発担当者が迷わずに実装できるよう、クラス構成、処理フロー、データ構造などの内部ロジックを具体的に記述することを目的とします。
# Oparation
詳細設計書は基本設計にて定義された内容をソースコードに落とし込むための手段を記載した文書です。  
基本設計書の内容が漏れなく部品レベルまで落とし込めている事が重要です。  
内容は各部品の名称を正確に記載し、誤解や実装漏れを防止できるよう記載します。  
実装する機能に対する技術が適切かを、容易性、速度、保守性、将来的な展開を意識して選定します。  
外部(DB)、内部(TBL)等、詳細に記載し、速やかに実装できるような配慮が必要です。
機能と物量を主な判断手段として分割します。  
1機能1項目が理想的ですが、複数に渡る場合もあります。
# Content
### 🟥 システム構成  
　　・OS  
　　・クラウド  
　　・DB  
　　・ネットワーク  
　　・ミドルウェア  
🟥 機能概要  
　システムが提供する機能とその処理の流れを整理します。  
　・機能一覧  
　・業務フロー  
　・処理フロー  
🟥 UI設計  
　機能を実行するための部品を選定し、適切な配置を行います。  
　UI(操作性、見た目)だけではなく、操作感(UX)も考慮します。  
　・画面構成  
　・画面遷移  
　・画面レイアウト  
　・入出力項目  
🟨 データ概要  
　システムで扱う主なデータと、その関連性や利用方法を整理します。  
　詳細なデータ型やテーブル構造は、詳細設計で定義します。  
　・主要データ一覧  
　・データ関連図  
　・データの保存・利用方法  
🟥 外部連携  
　連携手段を明確にし、利点のみではなく、問題点も記載してあると親切です。  
　・API  
　・CSV  
　・FTP  
　・ETL  
　・メール
